# 要件定義書

**プロジェクト名**: TOEIC 990 Vocabulary Trainer  
**バージョン**: 1.0  
**作成日**: 2026-05-30  
**作成者**: Iku

---

## 1. プロジェクト概要

### 背景
TOEIC L&R スコア 900 保持者が 990（満点）を目指すための、語彙・熟語特化型の自己学習Webアプリケーション。日々の学習で実際に使いながら開発し、将来的には同様の目標を持つ学習者にも公開する。

### 目的
- 公式TOEIC問題集・出る問特急から抽出した語彙・熟語を効率よく定着させる
- 間違えた問題を自動的に優先表示し、弱点を集中的に克服する
- 学習の継続状況を可視化してモチベーションを維持する

---

## 2. ユーザー定義

### プライマリユーザー
- TOEIC スコア 800〜950 を持つ上級学習者
- 満点（990）を目指している
- スマートフォン・PC 両方で学習したい

### ユーザーストーリー
| # | として | したい | なぜなら |
|---|--------|--------|----------|
| US-01 | 学習者 | 単語カードをめくって意味を確認したい | 隙間時間に素早く復習できるから |
| US-02 | 学習者 | 「覚えた／もう一度」で仕分けしたい | 苦手な単語に集中できるから |
| US-03 | 学習者 | 間違えた単語だけを再出題してほしい | 弱点を効率よく潰せるから |
| US-04 | 学習者 | 今日の学習進捗を確認したい | 達成感を得てモチベーションを保てるから |
| US-05 | 学習者 | 単語を自分で追加・編集したい | テキストの内容を自分でカスタマイズできるから |

## 3. 機能要件

### v1（MVP）対象機能
| ID | 機能名 | 優先度 | 説明 |
|----|--------|--------|------|
| F-01 | 単語カード表示 | Must | 単語・品詞・意味・例文をカード形式で表示 |
| F-02 | フリップアニメーション | Must | タップ/クリックで表裏を切り替え |
| F-03 | 覚えた／もう一度 仕分け | Must | 2択ボタンで学習ステータスを記録 |
| F-04 | 苦手単語の優先出題 | Must | 「もう一度」が多い単語を自動的に先に出す |
| F-05 | 進捗表示 | Must | 今日の正解数・残り枚数をヘッダーに表示 |
| F-06 | 単語の手動追加・編集・削除 | Must | 管理画面から単語データを操作できる |
| F-07 | セッション完了画面 | Should | 全カード終了時にスコアと達成メッセージを表示 |
| F-08 | カテゴリ別フィルター | Could | Part別・品詞別・テキスト別で絞り込み |

### v2 以降（将来検討）
- ユーザーアカウント機能（複数人対応）
- CSVインポートで単語を一括登録
- 出題モード選択（英→日 / 日→英 / 4択クイズ）
- 連続学習日数（ストリーク）表示

---

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブ対応 | スマートフォン（375px〜）・PC（1200px）両対応 |
| オフライン動作 | インターネット接続不要（ローカルストレージ使用） |
| データ永続化 | ブラウザの localStorage に保存 |
| パフォーマンス | カード表示まで1秒以内 |
| アクセシビリティ | キーボード操作対応（←→キーで仕分け） |
| ブラウザ対応 | Chrome / Safari / Firefox 最新版 |

---

## 5. 制約条件

- バックエンドなし（サーバー不要）
- 使用技術：HTML / CSS / JavaScript のみ
- ホスティング：GitHub Pages（無料）
- データ：localStorage（ブラウザ依存、端末をまたいだ同期は v1 対象外）
